# Template: React Router v7 Framework-Mode App Shell

Reference for scaffolding the framework-mode app shell when `routerMode == framework` and the files are
absent (offered by `fe-init` Step 4b, generated by `foundation-generator` / `integration-generator`).
Never overwrite existing files. Placeholders: `{baseDir}` (RR source root, e.g. `app/app`), `{appDir}`
(sibling of `package.json`, e.g. `app`).

**Path-base rule.** `react-router.config.ts` and the generated `.react-router/` types dir live in
`{appDir}`. `root.tsx`, `routes.ts`, `entry.server.tsx`, `entry.client.tsx` live under `{baseDir}`.

## `{appDir}/react-router.config.ts`

```ts
import type { Config } from '@react-router/dev/config'

export default {
  ssr: true,
  // Phase 1: only parameterless, loader-less routes may be prerendered (D5).
  // integration-generator maintains this array from plan.json pages[].rendering == "ssg".
  prerender: [
    // '/', '/about'
  ],
} satisfies Config
```

## `{baseDir}/routes.ts`

Central route config. Feature fragments are spread in (import + spread); integration-generator edits this
file. Route-module `file` paths are **app-root-relative** (never fragment-relative).

```ts
import { type RouteConfig, index, layout, route } from '@react-router/dev/routes'

// import { hotelSearchRoutes } from './features/hotel-search/routes'

  index('routes/home.tsx'),
  // ...hotelSearchRoutes,

```

## `{baseDir}/root.tsx`

`Layout` (document shell: `<Links/>`, `<Meta/>`, `<Scripts/>`, `<ScrollRestoration/>`), the root
`ErrorBoundary`, and — when `serverState == tanstack-query` — the `QueryClientProvider`.

```tsx
import { Links, Meta, Outlet, Scripts, ScrollRestoration } from 'react-router'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { useState } from 'react'

// Browser: a module-scope singleton is fine. Server render: a fresh client per request
// (a shared module-scope client would leak cache across concurrent SSR requests).
let browserQueryClient: QueryClient | undefined
function getQueryClient() {
  const make = () =>
    new QueryClient({ defaultOptions: { queries: { staleTime: 60_000 } } }) // D9 SSR default
  if (typeof document === 'undefined') return make() // server: always fresh
  return (browserQueryClient ??= make()) // browser: singleton
}

export function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <head><Meta /><Links /></head>
      <body>{children}<ScrollRestoration /><Scripts /></body>
    </html>
  )
}

export default function Root() {
  const [queryClient] = useState(getQueryClient)
  return (
    <QueryClientProvider client={queryClient}>
      <Outlet />
    </QueryClientProvider>
  )
}

export function ErrorBoundary() {
  // root-level error boundary; route modules add their own for scoped errors
  return <html><head><title>Error</title><Meta /><Links /></head><body>{/* ... */}<Scripts /></body></html>
}
```

> When `serverState == zustand-only`, drop the `QueryClientProvider` and the client helper — `Root`
> renders `<Outlet />` directly.

`server.listen()` runs before any loader. Also creates the per-request i18n instance (R2).

```tsx

  const g = globalThis as unknown as { __mswNodeStarted?: boolean }

// Fallback (D8): if the §8 dry-run interception assertion fails in some build,
```

## `{baseDir}/entry.client.tsx`

Client entry. **Only** the browser MSW worker bootstrap here — never the node hook.

```tsx
import { startTransition, StrictMode } from 'react'
import { hydrateRoot } from 'react-dom/client'
import { HydratedRouter } from 'react-router/dom'

async function main() {
  if (import.meta.env.DEV && import.meta.env.VITE_ENABLE_MOCKS === 'true') {
    const { worker } = await import('./mocks/browser')
    await worker.start({ onUnhandledRequest: 'bypass' })
  }
  // initialize client i18n from the serialized locale/resources, then:
  startTransition(() => {
    hydrateRoot(document, <StrictMode><HydratedRouter /></StrictMode>)
  })
}
main()
```

## `{baseDir}/routes/{name}.tsx` — route module (thin wrapper) shape

Per D10 the route module is a **thin wrapper**; the tested logic lives in an extracted page-body
component under `features/{feature}/pages/`.

```tsx
import type { Route } from './+types/{name}'
import { HotelSearchPage } from '~/features/hotel-search/pages/HotelSearchPage'
import { hotelSearchQuery } from '~/features/hotel-search/api/queries'
import { baseApiClient } from '~/lib/api.server' // D13 base client — loader-safe

export async function loader({ request }: Route.LoaderArgs) {
  const data = await baseApiClient.get(/* ... */)
  return { data, fetchedAt: Date.now() } // fetchedAt → initialDataUpdatedAt (D9 rule c)
}

export function meta({ data }: Route.MetaArgs) {
  return [{ title: /* t(titleKey) resolved server-side */ '' }]
}

export function HydrateFallback() { return <PageSkeleton /> }        // D10 loading
export function ErrorBoundary() { return <PageError /> }             // D10 error

export default function Route({ loaderData }: Route.ComponentProps) {
  return <HotelSearchPage {...loaderData} />                          // delegates to tested body
}
```

## Runtime

Built app runs locally via `npx react-router-serve ./build/server/index.js` (Node ≥ 22). Server-only
config from non-`VITE_` `process.env`. The plugin configures no deployment target (O3).
